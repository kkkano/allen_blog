## 索引基础概念

索引是数据库中用于提高查询性能的数据结构。就像书籍的目录一样，索引帮助数据库系统快速定位需要的数据。

### 为什么需要索引？

```sql
-- 没有索引的查询
SELECT FROM users WHERE username = 'john'; -- 全表扫描
-- 有索引的查询
CREATE INDEX idx_username ON users(username); -- 创建索引后性能显著提升
索引类型详解
1. B+Tree索引
最左匹配原则
最左匹配原则是指在复合索引中，索引按照定义顺序的第一个字段开始匹配。这意味着如果一个复合索引定义为(column_a, column_b)，那么column_a上的查询可以利用该索引，而单独对column_b的查询则不能直接利用此索引。

例子：

Sql
深色版本
CREATE INDEX idx_name_age ON users(name, age);
-- 查询name时可以使用索引
SELECT * FROM users WHERE name = 'John';
-- 查询name和age时也可以使用索引
SELECT * FROM users WHERE name = 'John' AND age > 30;
-- 仅查询age时无法有效使用索引
SELECT * FROM users WHERE age > 30;
B+Tree索引工作原理
B+Tree是一种平衡树数据结构，它允许快速查找、顺序访问、插入和删除操作。每个节点包含多个键值和指向子节点的指针。叶子节点存储了实际的数据行或指向这些行的指针。由于其特性，B+Tree索引非常适合范围查询。

2. 哈希索引
哈希索引通过计算索引列的哈希值来加速等值查询。尽管速度非常快，但它们不支持范围查询，并且对于频繁更新的列可能不太适合。

3. 全文索引
用于全文搜索，支持复杂的文本搜索功能，如布尔查询、模糊搜索等。适用于大型文本字段。

Sql
深色版本
CREATE FULLTEXT INDEX idx_fulltext ON articles(content);
4. 空间索引（R-Tree）
专门用于地理空间数据类型，支持复杂的空间关系查询。

Sql
深色版本
CREATE SPATIAL INDEX idx_spatial ON places(location);
