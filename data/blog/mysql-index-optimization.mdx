## 索引基础概念

索引是数据库中用于提高查询性能的数据结构。就像书籍的目录一样，索引帮助数据库系统快速定位需要的数据。

### 为什么需要索引？
sql
-- 没有索引的查询
SELECT FROM users WHERE username = 'john'; -- 全表扫描
-- 有索引的查询
CREATE INDEX idx_username ON users(username); -- 创建索引后性能显著提升
## 索引类型详解

### 1. B-Tree索引
最常用的索引类型，适用于全键值、键值范围和键前缀查询。
sql:allen_blog/data/blog/mysql-index-optimization.mdx
CREATE INDEX idx_name ON table_name (column_name);
### 2. 哈希索引
只适用于等值查询，但检索效率极高。

### 3. 复合索引
多列组合的索引，遵循最左前缀原则。
sql
CREATE INDEX idx_name_age ON users(name, age);

## 性能优化实践

### 索引优化案例
sql
-- 优化前
SELECT FROM orders
WHERE status = 'pending'
AND create_time > '2024-01-01';
-- 优化后
CREATE INDEX idx_status_create_time
ON orders(status, create_time);

1. 索引失效的场景

2. 如何处理大表索引
