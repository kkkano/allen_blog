<a name="2.3"></a>
### 2.3 线性回归基础

线性回归的目标是用一条直线来描述数据的关系。我们的直线方程是：

```
f(x) = w * x + b
```

- `x` 是城市人口（输入）。
- `f(x)` 是预测的利润（输出）。
- `w` 是斜率，表示人口对利润的影响大小。
- `b` 是截距，表示人口为 0 时的基准利润。

**我们的任务：**  
通过调整 `w` 和 `b`，让这条直线尽可能穿过所有的数据点。怎么衡量“尽可能好”呢？我们用一个叫**成本函数**的东西来计算预测值和实际值之间的误差。目标是让这个误差尽可能小。

**成本函数是什么？**  
简单来说，成本函数（记作 `J(w,b)`）是预测误差的平均值。误差越小，说明模型越好。我们会用**梯度下降法**来调整 `w` 和 `b`，一步步减小成本。

训练完成后，这条直线就能根据城市人口预测利润了。比如，给它一个新城市的人口数据，它就能告诉你餐厅可能赚多少钱。

<a name="2.4"></a>
### 2.4 计算成本函数

要用梯度下降优化模型，我们得先知道当前模型有多“好”或多“差”。这就需要成本函数来帮忙。成本函数的计算方法是：

1. 对每个样本，算出预测值 `f(x)` 和实际值 `y` 的差。
2. 把这个差平方（这样正负误差不会抵消）。
3. 把所有样本的平方差加起来，再除以样本数的两倍，得到平均误差。

<a name="ex01"></a>
#### 练习 1：实现成本函数

下面是计算成本的函数 `compute_cost`，请动手完成它：

```python
def compute_cost(x, y, w, b):
    """
    计算线性回归的成本。
    
    参数:
        x (ndarray): 形状 (m,)，输入数据（城市人口）
        y (ndarray): 形状 (m,)，实际值（利润）
        w, b (scalar): 模型参数（斜率和截距）
    
    返回:
        total_cost (float): 当前 w 和 b 下的成本
    """
    # 获取样本数
    m = x.shape[0]
    total_cost = 0
    
    ### 开始代码 ###
    cost_sum = 0
    for i in range(m):
        f_wb = w * x[i] + b  # 计算预测值
        cost = (f_wb - y[i]) ** 2  # 单样本的平方误差
        cost_sum += cost  # 累加所有误差
    total_cost = (1 / (2 * m)) * cost_sum  # 平均误差
    ### 结束代码 ###
    
    return total_cost
```

**代码详解：**
- `m` 是样本数，比如我们有 97 个城市。
- `f_wb` 是模型的预测值，用公式 `w * x[i] + b` 计算。
- `(f_wb - y[i]) ** 2` 是单个样本的误差平方。
- `cost_sum` 把所有误差加起来，最后除以 `2 * m`，得到平均成本。

**测试一下：**
用初始值 `w = 2` 和 `b = 1` 测试你的函数：

```python

initial_b = 1

print(f'初始 w 和 b 时的成本: {cost:.3f}')
```

**预期输出：**
```
初始 w 和 b 时的成本: 75.203
```

如果你的输出接近这个值，说明函数写对了！这个成本值告诉我们，当前模型的预测和实际数据差距还挺大，接下来要优化它。

<a name="2.5"></a>
### 2.5 梯度下降法

现在我们有了成本函数，但怎么让它变小呢？答案是**梯度下降法**。它就像爬山时找下坡路一样，通过计算“坡度”（梯度），一步步调整 `w` 和 `b`，让成本越来越小。

**梯度下降的步骤：**
1. 计算成本函数对 `w` 和 `b` 的“坡度”（专业术语叫偏导数）。
2. 用一个小的步子（学习率 `alpha`）沿着坡度下降的方向更新 `w` 和 `b`。
3. 重复这个过程，直到成本几乎不再下降。

**梯度是什么？**
- 对 `b` 的梯度：所有样本预测误差的平均值。
- 对 `w` 的梯度：所有样本预测误差乘以对应 `x` 的平均值。

<a name="ex02"></a>
#### 练习 2：实现梯度计算

下面是计算梯度的函数 `compute_gradient`，请完成它：

```python
def compute_gradient(x, y, w, b):
    """
    计算成本对 w 和 b 的梯度。
    
    参数:
        x (ndarray): 形状 (m,)，输入数据（人口）
        y (ndarray): 形状 (m,)，实际值（利润）
        w, b (scalar): 模型参数
    
    返回:
        dj_dw (scalar): 对 w 的梯度
        dj_db (scalar): 对 b 的梯度
    """
    m = x.shape[0]
    dj_dw = 0
    dj_db = 0
    
    ### 开始代码 ###
    for i in range(m):
        f_wb = w * x[i] + b  # 预测值
        dj_dw_i = (f_wb - y[i]) * x[i]  # w 的单样本梯度
        dj_db_i = f_wb - y[i]  # b 的单样本梯度
        dj_dw += dj_dw_i  # 累加 w 的梯度
        dj_db += dj_db_i  # 累加 b 的梯度
    dj_dw = dj_dw / m  # 平均 w 的梯度
    dj_db = dj_db / m  # 平均 b 的梯度
    ### 结束代码 ###
    
    return dj_dw, dj_db
```

**代码详解：**
- `f_wb` 是预测值，和成本函数一样。
- `dj_dw_i` 是每个样本对 `w` 的梯度，误差乘以 `x[i]`。
- `dj_db_i` 是对 `b` 的梯度，就是误差本身。
- 最后除以 `m`，取平均值。

**测试一下：**
用初始值 `w = 0` 和 `b = 0` 测试：

```python
initial_w = 0
initial_b = 0
dj_dw, dj_db = compute_gradient(x_train, y_train, initial_w, initial_b)
print('初始 w 和 b 时的梯度:', dj_dw, dj_db)
```

**预期输出：**
```
初始 w 和 b 时的梯度: -65.32884975 -5.83913505154639
```

这两个负值告诉我们，`w` 和 `b` 需要往正方向调整。接下来我们就用这些梯度来优化模型。

<a name="2.6"></a>
### 2.6 优化参数

有了成本函数和梯度计算，我们可以用梯度下降来找到最佳的 `w` 和 `b`。下面是完整的优化代码：

```python
def gradient_descent(x, y, w_in, b_in, cost_function, gradient_function, alpha, num_iters):
    """
    执行梯度下降优化参数。
    
    参数:
        x, y (ndarray): 数据
        w_in, b_in (scalar): 参数初始值
        cost_function: 计算成本的函数
        gradient_function: 计算梯度的函数
        alpha (float): 学习率（步长）
        num_iters (int): 迭代次数
    
    返回:
        w, b: 优化后的参数
    """
    w = w_in
    b = b_in
    
    for i in range(num_iters):
        dj_dw, dj_db = gradient_function(x, y, w, b)  # 计算梯度
        w = w - alpha * dj_dw  # 更新 w
        b = b - alpha * dj_db  # 更新 b
        
        if i % 150 == 0:  # 每 150 次打印一次成本
            cost = cost_function(x, y, w, b)
            print(f"迭代 {i:4}: 成本 {cost:.2f}")
    
    return w, b
```

**关键点：**
- `alpha` 是学习率，控制每次调整的步子大小。太大会跳过头，太小会很慢。
- `num_iters` 是迭代次数，决定我们走多少步。

**运行优化：**
用初始值 `w = 0`、`b = 0`，学习率 `alpha = 0.01`，迭代 1500 次：

```python
initial_w = 0.
initial_b = 0.
iterations = 1500
alpha = 0.01

w, b = gradient_descent(x_train, y_train, initial_w, initial_b,
                        compute_cost, compute_gradient, alpha, iterations)
print("优化后的 w, b:", w, b)
```

**预期输出：**
```
迭代    0: 成本 6.74
迭代  150: 成本 5.31
...
优化后的 w, b: 1.16636235 -3.63029143940436
```

成本逐渐减小，说明模型在变好。最终的 `w` 和 `b` 就是我们想要的直线参数。
